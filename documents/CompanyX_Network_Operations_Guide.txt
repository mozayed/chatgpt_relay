SECTION 3: PORT TROUBLESHOOTING PROCEDURES
================================================================================

SYMPTOM: Port showing down status

Step 1: Physical Layer Check
- Verify cable is properly seated on both ends
- Check for visible cable damage
- Test with known good cable if available
- Verify link lights on switch and end device

Step 2: Configuration Check
Run these commands on the switch:
- show interface [interface-id] status
- show interface [interface-id] 
- show running-config interface [interface-id]

Common issues:
- Port administratively shutdown: Use "no shutdown" command
- Speed/duplex mismatch: Set to "auto" on both sides
- VLAN mismatch: Verify access VLAN matches end device

Step 3: Port Security
Check if port security is blocking:
- show port-security interface [interface-id]
- If violation, clear with: clear port-security sticky [interface]

Step 4: Spanning Tree
Verify port is not in blocking state:
- show spanning-tree interface [interface-id]
- If blocking, check for loops in topology

Step 5: Error Counters
Check for errors:
- show interface [interface-id] | include error
- High CRC errors = bad cable or NIC
- Input errors = usually physical layer issue

SECTION 4: WIFI CONNECTIVITY ISSUES
================================================================================

SYMPTOM: Users cannot connect to WiFi

First Response (5 minutes):
1. Check wireless controller status: https://wlc.companyx.local
2. Verify AP is online in controller GUI
3. Check SSID is broadcasting
4. Verify DHCP scope has available addresses

Common Causes:
- AP lost power (check PoE switch port)
- Controller authentication server down (check RADIUS)
- DHCP scope exhausted (expand range)
- AP firmware mismatch (update to controller version)

Access Point Naming:
Format: FLOOR-LOCATION-APxx
Example: 04-EAST-AP01 (4th floor, east wing, AP #1)

Reboot Access Point:
From controller: Wireless > Access Points > Select AP > Reboot

Physical Reboot:
Locate switch port feeding AP and bounce the port:
- interface gigabitethernet x/x/x
- shutdown
- (wait 10 seconds)
- no shutdown

SECTION 5: INTER-VLAN ROUTING ISSUES  
================================================================================

SYMPTOM: Devices in different VLANs cannot communicate

Troubleshooting Steps:

1. Verify SVI (Switch Virtual Interface) is up:
   show ip interface brief | include Vlan
   
2. Check routing:
   show ip route
   Verify routes exist for both VLANs

3. Verify IP helper-address for DHCP:
   show running-config interface vlan [id]
   Should see: ip helper-address [DHCP-server-IP]

4. Check firewall rules:
   Review ACLs between VLANs
   show ip access-lists

5. Test with ping:
   ping [destination-IP] source [source-interface]

Remember: Our core switches (HQ-CORE-SW01/02) handle all inter-VLAN routing.
Access switches should have "ip default-gateway" pointing to core.

SECTION 6: DHCP TROUBLESHOOTING
================================================================================

SYMPTOM: Device not receiving IP address

Primary DHCP Server: 10.0.1.10 (Windows Server 2022)
Secondary DHCP Server: 10.0.1.11 (Failover pair)

Quick Checks:
1. Verify DHCP scope has available leases
2. Check server services are running
3. Verify ip helper-address on VLAN interface
4. Check for rogue DHCP servers (show dhcp snooping binding)

DHCP Scopes by VLAN:
- VLAN 20 (IT): 10.0.20.100 - 10.0.20.200
- VLAN 30 (Sales): 10.0.30.100 - 10.0.30.200  
- VLAN 40 (HR): 10.0.40.100 - 10.0.40.200
- VLAN 50 (Guest): 10.0.50.100 - 10.0.50.250

Lease Time: 8 hours (corporate), 1 hour (guest)

Force Release/Renew on Client:
Windows: ipconfig /release && ipconfig /renew
Linux: sudo dhclient -r && sudo dhclient
Mac: System Preferences > Network > Renew DHCP Lease

SECTION 7: CDP/LLDP NEIGHBOR DISCOVERY
================================================================================

Use CDP to identify connected devices:

Show all neighbors:
show cdp neighbors

Show detailed neighbor info:
show cdp neighbors detail

Show specific interface:
show cdp neighbors [interface-id] detail

Useful for:
- Identifying what's plugged into a port
- Finding IP address of connected device
- Verifying correct device is connected
- Network topology mapping

Note: CDP is Cisco proprietary. For non-Cisco devices, use LLDP:
show lldp neighbors
show lldp neighbors detail

SECTION 8: PORT CONFIGURATION STANDARDS
================================================================================

Standard Access Port Configuration:

interface GigabitEthernet1/0/1
 description [Department]-[Room]-[Jack#]
 switchport mode access
 switchport access vlan [XX]
 spanning-tree portfast
 spanning-tree bpduguard enable
 no shutdown

Standard Trunk Port Configuration:

interface GigabitEthernet1/0/48
 description TRUNK-TO-[DEVICE]
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,40,50
 no shutdown

Port Security Template (for sensitive areas):

interface GigabitEthernet1/0/10
 switchport port-security
 switchport port-security maximum 2
 switchport port-security violation restrict
 switchport port-security mac-address sticky
